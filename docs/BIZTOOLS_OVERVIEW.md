### Color Palette

**Professional & Trust-Building:**
- Primary: #2563EB (Blue 600) - buttons, links, accents
- Secondary: #059669 (Emerald 600) - success states, positive values
- Warning: #F59E0B (Amber 500) - alerts, important notes
- Error: #DC2626 (Red 600) - validation errors
- Neutral: #64748B (Slate 500) - text, borders
- Background: #F8FAFC (Slate 50) - page background
- Card: #FFFFFF - calculator containers

### Typography

**Font Stack:**
- Headings: Inter, system-ui, sans-serif (600-700 weight)
- Body: Inter, system-ui, sans-serif (400-500 weight)
- Numbers: 'JetBrains Mono', monospace (calculator outputs)

**Sizes:**
- H1: 2.5rem (40px) - tool name
- H2: 1.875rem (30px) - section headers
- H3: 1.5rem (24px) - subsections
- Body: 1rem (16px) - default text
- Small: 0.875rem (14px) - labels, helper text
- Output: 1.5-2rem (24-32px) - calculated results

### Component Patterns

**Input Fields:**
```

**Output Cards:**
```
┌────────────────────────┐

**Buttons:**
- Primary: Blue bg, white text, shadow

- Hover: Slight darken, lift

### Responsive Behavior

**Desktop (>1024px):**
- Two-column layout (input left, output right)
- Calculator max-width: 1200px
- Sidebar for ads (right side)

**Tablet (768-1023px):**
- Two-column maintained
- Reduced padding
- Sidebar below calculator

**Mobile (<768px):**
- Single column (input, then output)
- Full-width cards
- Larger touch targets (min 44px)
- Sticky calculate button at bottom

## Technical Stack

### Frontend
- **Framework:** Next.js 14 (App Router)
- **Language:** TypeScript
- **Styling:** Tailwind CSS
- **Forms:** React Hook Form
- **Validation:** Zod
- **Charts:** Recharts
- **Icons:** Lucide React

### Calculators
- **Logic:** Pure functions, separate from components
- **Testing:** Jest + React Testing Library
- **Formatting:** Intl.NumberFormat, date-fns
- **State:** useState for simple, useReducer for complex

### SEO
- **Metadata:** generateMetadata per page
- **Schema:** JSON-LD for FAQ, HowTo, WebApplication
- **Sitemap:** Auto-generated from calculator list
- **Analytics:** Plausible or Google Analytics 4

### Performance
- **Images:** Next.js Image component
- **Fonts:** next/font (Inter preloaded)
- **Loading:** Suspense boundaries
- **Caching:** Static generation where possible

## Risk Mitigation

### SEO Risks
- **Algorithm changes** - Diversify traffic sources (Reddit, Product Hunt)
- **Competitor clones** - Move fast, build 15+ tools quickly
- **KD increases** - Target easier terms first, build authority
- **Google preference for brands** - Use exact-match domain, build trust signals

### Technical Risks
- **Calculation errors** - Extensive testing, user feedback loop
- **Performance issues** - Static generation, optimize images
- **Mobile problems** - Mobile-first development approach

### Business Risks
- **Low traffic** - Have 15+ tools for aggregated traffic
- **Ad blocker usage** - Add affiliate links as backup revenue
- **Low RPM** - Target high-CPC terms (freelance, consulting)
- **Time investment** - Reusable components, calculator templates

## Notes & Learnings from BuildCalc

### What Worked
1. **Two-column layout** - Clear separation of input/output
2. **Real-time calculation** - No submit button needed
3. **Educational content** - Below-calculator SEO content
4. **Author attribution** - E-E-A-T signals
5. **Schema markup** - FAQ rich results

### Improvements for BizTools
1. **Faster development** - Reusable component library
2. **Better mobile UX** - Single column on mobile, larger buttons
3. **Save/share features** - URL params, localStorage
4. **More internal linking** - Related calculators grid on every page
5. **Comparison mode** - Side-by-side scenarios for rate calculators

### Technical Lessons
1. **Input validation** - Use Zod schemas
2. **Number formatting** - Intl.NumberFormat for currency/percentages
3. **Responsive images** - Next.js Image component
4. **Font loading** - Preload Inter with next/font
5. **Analytics** - Track calculator completions as goals
